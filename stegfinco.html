How StegPay is wired in

StegPay is deliberately narrow: it verifies that a payment event happened. It does not decide what that payment authorizes.

Stripe or external payment event → StegPay verifies event integrity and required metadata → StegPay emits deterministic payment evidence → StegFinCo treats the event as evidence only → StegFinCo checks current authority and admissibility → ALLOW / DENY / FAIL_CLOSED → receipt written → downstream action may proceed only if admitted

Boundary doctrine

Payment is evidence. Governance creates authority. StegFinCo controls effect. Fin-Co constrains interpretation. Support payment evidence may be recorded. Support payment entitlement must not be created. Uncertainty fails closed.
