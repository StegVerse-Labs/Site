## Authenticated review intake

External Chat now submits the same canonical cooperative-review package generated by the browser. The user must explicitly opt in, provide at least one review-scope item, and enter a review access token.

The access token:

```text
is held only in current page memory
is sent only in the Authorization header
is cleared after success or failure
is not written to browser storage
is not placed in the package or receipt
is not published to the wiki
```

The service derives an append-only package ID from the compatibility receipt and submission hash. It stores only the review package; raw framework artifacts remain prohibited.

Successful intake returns:

```text
package_id
external-review-intake-receipt:hmac-sha256:...
review_state = AWAITING_DELEGATED_REVIEW
raw_submission_stored = false
wiki_record_created = false
publication_authorized = false
standing_created = false
```


Reviewer authentication requires:

```text
reviewer_ref present in the configured registry
SHA-256 match for the supplied reviewer token
current valid_from / valid_until window
delegation_ref present
requested field scopes delegated
publication_review scope when publication review was requested
```

Reviewer delegation does not grant certification, publication, execution, or standing authority.

## Append-only storage and conflict behavior

SQLite review storage maintains separate append-only tables for review packages and correction receipts.

```text
same package identity + same content -> idempotent return
same package identity + different content -> 409 conflict
same challenged receipt + same correction -> idempotent return
same challenged receipt + different correction -> 409 conflict
identity drift in challenged receipt or submission hash -> fail closed
expired or out-of-scope delegation -> fail closed
```

A valid reviewer action produces an HMAC-bound `external-framework-correction-receipt` and still returns:

```text
publication_authorized = false
certification_created = false
standing_created = false
```

## Sharing posture

External Chat is a bounded compatibility-intake and authenticated cooperative-review prototype with receipted catalog synchronization, package-only storage, delegated reviewer checks, append-only correction receipts, and conflict detection. It is not framework certification, automatic publication, or proof of general interoperability.
