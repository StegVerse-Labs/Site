Operator-local continuation · authenticated browser session · no prompt injection

Open a specific ChatGPT session
without publishing its private URL.

This Site surface stores one ChatGPT conversation link in this browser only. It can reopen that authenticated conversation in web ChatGPT. It cannot submit a message, control the ChatGPT page, transfer account access, or grant StegVerse execution authority.

Boundary: the conversation URL is never sent to StegVerse, committed to GitHub, embedded in page source, or included in a Site receipt. Browser-local storage is convenience state, not custody, authentication, authority, or durable ecosystem persistence.

What this does and does not do

  • Opens only an HTTPS URL on chatgpt.com whose path begins with /c/.
  • Requires the operator to already be authenticated to the ChatGPT account that owns the conversation.
  • Does not place the private conversation identifier in repository code, query parameters, analytics, or Site-generated records.
  • Does not inject, prefill, or submit a prompt into the conversation.
  • Does not treat opening a browser tab as execution, continuity, custody, admissibility, or activation evidence.
