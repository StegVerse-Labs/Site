StegVerse Ecosystem Chat
ask once, route safely.

This page has one public purpose: let a user describe an ecosystem request and see how StegVerse classifies it before any real execution. It is a governed chat preview, not a shell, not a repo control panel, not a proof issuer, and not the full product.

Try the governed chat preview How the boundary works
Boundary: Site can draft, classify, and display. It cannot execute shell commands, access credentials, modify repositories, issue authority receipts, or grant admissibility. Site fields generate local preview text only until a governed backend, authority check, rate limits, and receipt path exists.

What happens here

The page follows a single public path instead of multiple competing entry points.

1. Describe requestExample: “summarize the current Site state” or “classify this transition request.”
2. Local classificationThe browser labels the route, transition intent, task status, authority boundary, and restricted-admin risk.
3. Preview onlyThe result is a local scaffold. No proof receipt, no repo write, no provider call, no shell access.
4. Continue to destinationThe contextual continuation panel offers a governed transition destination without granting authority.

Ecosystem LLM routing bands

The full product should expose the source class and micronode load for every interaction. This preview computes a local, non-authorizing visualization so users can see whether a request is intra-ecosystem, inter-ecosystem, research, provider, solver, or receipt-heavy.

IntraStegVerse repos, wikis, manifests, receipts, standards, and local ecosystem records.
InterConnected adapters, partner systems, provider clients, and external ecosystem nodes.
ResearchNetworked sources outside the ecosystem when current outside evidence is necessary.
ProviderLLM provider calls, model routing, fallback, cost, latency, and quota accounting.
SolverMath-problem solver, calculation traces, symbolic checks, unit conversion, and proof-step verification.
ReceiptHash, replay, reconstruction, admissibility, continuity, and authority evidence load.

Preview-only: values are deterministic local classifications, not live telemetry, not network access, and not authority-issued proof.

Public guardrails

Ecosystem Chat is not a public terminal. It is an intent-to-governed-action interface. Public users can request advancement; restricted operators and authorized ecosystem entities determine whether a task may execute through an approved task definition and auditable receipt path.
No raw shellMessages are routed as requests. Arbitrary bash, GitHub CLI, token inspection, workflow edits, and destructive commands are not accepted here.
Allowed tasks onlyExecutable tasks must be defined outside this page by governed standards, manifests, workflows, and reviewable scripts.
Authority checkedRestricted admin requests such as branch deletion, secret rotation, permissions, protected settings, or release actions require separate authority.
Receipts expectedAccepted executable transitions must return status and receipt data from the governing authority layer.
