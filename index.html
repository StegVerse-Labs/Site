Audit is evidence. Commit-time admissibility is control.

Logs can explain what happened. A commit boundary determines whether an action is still valid before it becomes an irreversible effect.

The StegVerse pattern is simple: reconstruct current state, re-derive current permission, recompute admissibility, then allow, deny, or fail closed.
